# Frontend Design Prompt — House Rental Management System

## Project Overview
A Django-based **Property & Tenant Management System** for managing rental properties, tenants, tenancies, reservations, and financial operations. The interface serves **Property Owners, Managers, and Admins**.

---

## Design Goals
✅ **Professional & Trustworthy** — Clean, minimal UI with clear hierarchy  
✅ **Modern & Accessible** — Contemporary design patterns; WCAG compliant  
✅ **Efficient & Fast** — Reduces cognitive load; quick decision-making  
✅ **Responsive & Mobile-Friendly** — Works on mobile, tablet, desktop  
✅ **Data-Rich Dashboards** — Clear visualization of key metrics and operations  

---

## Component Design Standards

### Sidebar Navigation
- Fixed width 240px, dark background
- Clear section labels (uppercase, muted)
- Active/hover states with brand color
- Badge notifications (red pill on nav items)
- User card at footer with avatar, name, role

### Top Bar
- Sticky, fixed height (58px)
- Page title on left, actions on right
- Notification and settings buttons
- Subtle bottom border

### Cards & Containers
- White background, 1px subtle border
- Consistent padding (20px body, 16px headers)
- Rounded corners (10-14px)
- Light shadow for depth

### Stat Cards
- Grid layout (4 per row on desktop, responsive)
- Icon with colored background
- Large numeric value (24px bold)
- Label and trend indicator (↑↓)
- Hover effect: slight shadow increase

### Tables
- Clean header row with light background
- Striped rows (hover background: #FAFBFA)
- Proper cell padding (13px vertical, 16px horizontal)
- Sortable column headers (indicate with small icon)

### Buttons
- **Primary**: Brand green, white text
- **Secondary**: White with subtle border
- **Danger**: Red background, red text on light red
- Consistent padding; smooth transitions

### Forms
- Block-level labels with required indicator (red *)
- Full-width inputs with focus state
- Subtle help text below each field
- 2-3 column grid layouts on desktop, stack on mobile

### Alerts & Messages
- Colored left border (3px) matching status
- Colored background (light) matching status
- Icon + message + close button
- Auto-dismiss after 5 seconds

### Empty States
- Centered icon (36px, 40% opacity)
- Short heading + descriptive text
- Optional action button
- Light gray color palette

## Key Features to Polish

### 1. **Visual Hierarchy**
- Large, bold headings (22px, 600wt)
- Secondary headings (16px, 600wt)
- Body text (14px, 400wt) with 1.6 line-height
- Labels (13px, 500wt)
- Help text (12px, muted color)

### 2. **Interactive States**
- Buttons: hover effect (color shift), active (deeper shade)
- Links: underline on hover, color change
- Inputs: focus ring (3px colored shadow), border color change
- Cards: subtle shadow increase on hover

### 3. **Data Visualization**
- Progress bars with brand colors
- Status badges (inline-flex, compact sizing)
- Icons with semantic colors
- Sparklines or small charts where relevant

### 4. **Spacing & Alignment**
- Consistent grid (8px baseline)
- Margins between sections (24px vertical)
- Card padding (20px)
- Form field spacing (18px between groups)

### 5. **Typography**
- Font: 'DM Sans' (Google Fonts)
- Font stack: `'DM Sans', sans-serif`
- Weights: 300, 400, 500, 600 (no heavier needed)
- Letter-spacing for labels: 0.08em uppercase

### 6. **Mobile Responsiveness**
- Breakpoint: 800px
- Sidebar: hidden, toggle button on mobile
- Grid layouts: 1 column on mobile
- Padding reduced on mobile (16px vs 28px)
- Touch-friendly buttons (min 44x44px)

## Design Principles to Follow

1. **Consistency** — Use the defined color palette, spacing, and typography throughout
2. **Clarity** — Clear labels, proper hierarchy, obvious interactions
3. **Feedback** — Always confirm user actions (success messages, loading states)
4. **Affordance** — Make interactive elements look clickable (buttons, links)
5. **Simplicity** — Remove clutter; focus on essential information
6. **Efficiency** — Minimize clicks; show related actions together
7. **Accessibility** — Sufficient color contrast, semantic HTML, keyboard support
